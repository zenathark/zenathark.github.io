3 Excercises

  1. Open and explore a basic terminal emulator. The command line interface is one of the most primitive forms of interaction with an operating system. Most tools for application development have ways of intercation through the command line. In order to use the command line you need to open a program called Terminal Emulator. For windows, there are two terminal emulator built-in. The one to be used for this course is called cmd.exe. As your firs excercise, open the program cmd.exe. There are two different ways of invoking the command:

    1. For windows 8-10, press the Windows Key and type CMD, this will sort all programs on the menu beeing the first the terminal emulator.
    2. Press the Windows Key and at the same time press R. This will pop up a dialog asking for the name of the program to execute. Type in CMD and click Accept or press the enter key.

  2. Close the terminal emulator. Closing the emulator can be done through two methods:

    1. Click on the cross button located on the top right corner of the terminal emulator window.
    2. With the window focus over the emulator, write exit and press the Enter Key. Example:
    exit

    Include a screenshot of the open emulator with the command written in it before pressing the Enter key in the report.

  3. Understanding the directory structure. Operating Systems sort its content using two types of data: files and directories (or folders). Directories are the most basic data type. Directories are used to contain files. Directories are arranged using a tree structure. The base of the tree is the letter assigned to the drive. Usually when you open the terminal emulator, it shows something like this: 

    C:\WINDOWS\system32>

    The first letter is the drive your are on. Any command written on the command line will have an effect over the C drive. On Windows systems, the C Drive contains the Windows Operating System and is contained on the WINDOWS directory. Please be carefull when writing commands over that drive. The following part of the structure show in the example is the active directory. In the example, the active directory is the system32 directory that is inside of the WINDOWS directory.

    In this excersise, you must identify which directory is active when you open the terminal emulator and on which drive you are on.

  4. Moving through directories and Managing. The command for moving through directories is CD, an acronym of Change Directory. Write the next commands on the terminal emulator and write down the results shown by the terminal.

    • Moving backwards 

      cd ..

    • Going to HOME 

      cd

    • Creating a directory 

      mkdir practica1

    • Displaying the content of the active directory 

      dir

    • Moving forward 

      cd forward

    • Going back again 

      cd ..

    • Deleting a directory 

      rmdir practica1

    Include a screenshot of the terminal with all the commands executed.

  5. Creating a directory structure. Create a directory structure contained inside of HOME and write down the output of the terminal.

    • Create a parent directory called Directorio1
    • Create three directories called subdirectorio1, subdirectorio2, subdirectorio3 inside Directorio1
    • Create a directory called subsubdirectorio1 inside subdirectorio2

    After executing all the commands, include in your report the output of the following command with theme HOME directory active: 

    tree /F /A .
